What does it do?

Think of the global.R file as a file that is being run once before your app starts. That means you can use it for all sorts of data processing, running models, and, of course, to load in your data.

{{< hl-text orange >}} Any R objects that are created in the global.R file become available to the app.R file, or the ui.R and server.R files respectively! {{< /hl-text >}} This makes building a dashboard a lot cleaner. Now, you can handle a lot of your pre-processing in the global.R file and then feed the results into the server.R file simply by using the object. No need to use source() to load anything either.
